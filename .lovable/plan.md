# Step 1 — 뼈대: 입장과 시간표

PRD의 "한 번에 한 Step" 원칙에 따라 이번 턴은 Step 1만 구현합니다. Grill Me, PRD 폼, 리뷰, 슬라이드 생성기 등은 이후 Step에서 다룹니다.

## 완료 기준 (PRD Step 1 그대로)
- 두 브라우저에서 각각 참가자·강사로 입장했을 때 서로 다른 홈 화면이 보인다.
- 새로고침해도 로그인이 유지된다.

## 구현할 것

### 1) 백엔드 (Lovable Cloud 활성화)
- `sessions` — 강사가 발급한 입장 코드 관리 (participant_code, instructor_code)
- `users` — 닉네임 + 역할(participant/instructor) + session_id. 이메일 없음.
- `session_state` — 현재 열린 스테이지 번호 (Step 2에서 사용, 스키마만 미리)
- RLS: 참가자는 같은 세션 참가자만 조회, 강사는 자기 세션 전체 조회. 로그인은 "코드+닉네임" 확인 후 클라이언트에 user_id를 로컬 저장(입장 코드 = 세션 비밀). 이메일/비밀번호 인증은 사용하지 않음 — PRD 제약.

### 2) 로그인 화면 `/` 
- 카드 UI: "입장 코드"(6자리), "닉네임" 입력, "입장" 버튼
- 서버 함수 `enterSession`: 코드 검증 → 역할 판정 → 닉네임 중복 체크 → users insert → user_id 반환
- localStorage에 `user_id`, `session_id`, `role` 저장 (세션 지속)
- 강사 코드로 들어오면 강사 홈, 참가자 코드는 참가자 홈으로 라우팅

### 3) 참가자 홈 `/home`
- 상단 **명찰(nametag)** 컴포넌트: 닉네임 + 경로 자리(추후 Step 11에서 채움) + 로그아웃
- 6교시 **시간표 카드 6장** (S1~S6): 교시명, 목표 1줄, 상태(잠김/열림/완료), 도장 자리
- Step 1에서는 S1만 "열림", S2~S6 자물쇠 아이콘. 클릭 시 "아직 준비 중" 토스트.

### 4) 강사 홈 `/instructor`
- 명찰: "강사 · {닉네임}"
- 세션 정보 카드: 참가자 입장 코드 표시(공유용 큰 글씨), 접속 참가자 수
- 6교시 시간표 카드(참가자와 동일 레이아웃, 상단에 "강의 모드" 자리 — Step 3에서 활성화)
- 접속자 리스트(닉네임만) — 실시간 갱신은 Step 2에서 강화

### 5) 학교 테마
- `styles.css`의 CSS 토큰 재정의:
  - `--primary`: 칠판 그린 #2F6B4F
  - `--accent`: 옐로 #FFC94A
  - `--destructive`: 코랄 #FF6B57
  - 배경 화이트, 카드 미세한 종이 질감(border-radius 크게)
- 폰트: `@fontsource/gowun-dodum` (한글 친화 부드러운 손글씨체) + `@fontsource/noto-sans-kr` 본문
- 상태 표시는 색+아이콘+텍스트 병행(도장 = Stamp 아이콘 + "완료" 라벨)
- 이모지 배제, 성인 학습자용 밝고 차분한 톤

### 6) 라우팅
- `/` — 로그인 (localStorage에 세션 있으면 역할별 홈으로 자동 리다이렉트)
- `/home` — 참가자 홈 (참가자 아니면 `/`로)
- `/instructor` — 강사 홈 (강사 아니면 `/`로)
- `__root.tsx` head: 한국어 title/description으로 교체

### 7) 시드 데이터 (마이그레이션)
- 기본 세션 1개 자동 생성: 참가자 코드 예 `SPOON1`, 강사 코드 예 `TEACH1` — 첫 실행에서 바로 시연 가능.

## 기술 세부
- TanStack Start + 서버 함수(`createServerFn`)로 로그인/조회
- 익명 인증이 아니라 "코드+닉네임" 방식이므로 Supabase Auth는 사용하지 않고, RLS는 서비스 롤 서버 함수로 우회 + 서버 함수 내부에서 세션 소속 검증
- 새 public 테이블에는 GRANT + RLS + 정책 명시
